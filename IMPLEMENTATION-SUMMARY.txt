## 🔐 How Authentication Works

### Registration Flow:
1. User enters name, email, password
2. Backend hashes password with bcrypt
3. Saves user to MongoDB
4. Returns JWT token
5. Frontend stores token in localStorage
6. User is logged in

### Login Flow:
1. User enters email, password
2. Backend finds user in database
3. Compares password hash
4. Returns JWT token if valid
5. Frontend stores token
6. Loads user-specific data

### Protected Requests:
1. Frontend sends token in Authorization header
2. Backend middleware verifies token
3. Extracts user ID from token
4. Allows access to user's data only

## 📊 Database Schema

### Users Collection
```javascript
{
  _id: ObjectId,
  name: String,
  email: String (unique),
  password: String (hashed),
  role: String,
  createdAt: Date
}
```

### Orders Collection
```javascript
{
  _id: ObjectId,
  userId: ObjectId (ref: User),
  orderId: String,
  customer: String,
  product: String,
  price: String,
  date: String,
  status: String,
  source: String,
  createdAt: Date
}
```

### Customers Collection
```javascript
{
  _id: ObjectId,
  userId: ObjectId (ref: User),
  name: String,
  email: String,
  status: String,
  joined: String,
  source: String,
  createdAt: Date
}
```

### Tasks Collection
```javascript
{
  _id: ObjectId,
  userId: ObjectId (ref: User),
  taskId: String,
  title: String,
  status: String (todo/progress/done),
  tag: String,
  createdAt: Date,
  updatedAt: Date
}
```

## 🔧 API Endpoints Summary

### Public (No Auth Required)
- POST `/api/auth/register` - Create account
- POST `/api/auth/login` - Login

### Protected (Token Required)
- GET `/api/auth/me` - Current user
- GET `/api/orders` - User's orders
- POST `/api/orders/bulk` - Bulk create orders
- GET `/api/customers` - User's customers
- POST `/api/customers/bulk` - Bulk create customers
- GET `/api/tasks` - User's tasks
- POST `/api/tasks` - Create task
- PUT `/api/tasks/:id` - Update task
- DELETE `/api/tasks/:id` - Delete task
